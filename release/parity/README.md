# release/parity/ (SDK)

SDK-perspective parity attestations, generated by
`scripts/release_run_parity.py`. Drives the runtime-side parity
scripts (in the sibling `octomil-runtime` checkout) via subprocess,
captures the runtime artifacts, and emits SDK-prefixed mirrors plus a
top-level `sdk_release_parity_summary.json`.

Layout (post-run):

```
release/parity/
├── audio.transcription/
│   ├── audio.transcription.parity.json        (runtime mirror)
│   ├── audio.transcription.parity.report.txt  (runtime mirror)
│   └── sdk_audio.transcription.json           (SDK perspective)
├── audio.vad/
│   ├── audio.vad.smoke.json
│   ├── audio.vad.smoke.report.txt
│   ├── _no_python_reference.md
│   └── sdk_audio.vad.json
├── audio.speaker.embedding/
│   ├── audio.speaker.embedding.smoke.json
│   ├── audio.speaker.embedding.smoke.report.txt
│   ├── _no_python_reference.md
│   └── sdk_audio.speaker.embedding.json
└── sdk_release_parity_summary.json
```

All generated files are gitignored; only this README is tracked. The
schema (`release/parity/_schema.json`) lives in `octomil-runtime` —
the SDK references it by path during validation.

Usage:

```bash
# sibling-checkout layout (default)
python3 scripts/release_run_parity.py

# explicit runtime path
OCTOMIL_RUNTIME_REPO=/abs/path/to/octomil-runtime \
    python3 scripts/release_run_parity.py
```

See `octomil-runtime/docs/release/parity-gates.md` for what the
gates measure and how to interpret a fail.
